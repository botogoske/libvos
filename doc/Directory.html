Directory module is an interface to file system. When File module is for reading and writing content of file in file system, Directory module is created for handling content of directory: list of files with their attributes.

Directory module is built using two class: DirNode and Dir. DirNode represent each node (node is a single directory/file/symbolic link.) in the content of directory, it contains node name, permission, size, modification time, and attribute modification time.

Dir class is a class that collecting and processing all the node in one directory, including all the nodes in the subdirectory, into one root DirNode object.

DirNode Variables

TypeNameDescription
int _mode Type and permission of node.
int _uid User ID of node in file system.
int _gid Group ID of node in file system.
long _size Size of node.
long _mtime Last modification time to the contents of file.
long _ctime Last modification time to the attributes of node.
Buffer _name Name of node.
Buffer _linkname A real path if node is a symbolic link.
DirNode* _next Pointer to the next node in the same directory.
DirNode* _prev Pointer to the previous node in the same directory.
DirNode* _child Pointer to the child node if this node is a directory.
DirNode* _link Pointer to the real node object if this node is a symbolic link to directory with the same root.
DirNode* _parent Pointer to the parent directory.

Parameter node : a pointer to DirNode object that will be updated.

Dir::get_node

Code DirNode* get_node(Buffer* path, const char* root, int root_len)
Parameter path : path to directory.
root : the root directory of this Dir object.
root_len : length of string root.
Return Value DirNode* : success, pointer to DirNode object of path.
NULL : fail.
Description Get pointer to DirNode object pointed by path. This method is used to simplified and minimize memory usage of directory caches by using link to real node if node is a symbolic link in the same root.
Case example, 
	0 | /root/dir
	x | /root/dir/real/path/to/dir
	...
	node | linkname | /root/dir/real/path/to/dir
0 is a root directory, the first node in the DirNode object. path is a symbolic link to x, so, instead of creating list of directory path again, we just point the _link of path node to the same value of child pointer of directory x.
